How to check dependencies / packages

Instruction: verify required packages and versions

What this is

The server installation may require specific system packages, runtimes, or versions (e.g. Java, Python, libs). Missing or wrong versions cause install or runtime failures.

How to do it

  1. Read the server’s installation or prerequisites doc and note every required package and version.
  2. On Linux: use rpm -q <package> or dpkg -l <package>, or which <cmd> and <cmd> --version for runtimes.
  3. Install missing packages with the distro package manager (yum/dnf/apt) or follow the vendor’s guide for third-party runtimes.
  4. Re-run the installer after satisfying dependencies.
