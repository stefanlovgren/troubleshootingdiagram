How to check service / run-as user

Instruction: see which user the service runs as

What this is

Permission errors often happen because the process runs as a different user than you expect. You need to know which user (and group) the server process runs as so you can fix ownership of files and dirs.

How to do it

  1. Linux (systemd): systemctl show <unit> --property=User --property=Group to see the user/group of the service.
  2. Linux (process): ps -o user= -p <PID> or ps aux | grep <app> to see the user of a running process.
  3. Ensure config files, data dirs, and log dirs are readable (and writable where needed) by that user. Use chown / chmod as needed.
