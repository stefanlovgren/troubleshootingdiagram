How to check bind address / interface

Instruction: verify which address or interface the service binds to

What this is

A service can listen on 0.0.0.0 (all interfaces), 127.0.0.1 (localhost only), or a specific IP. If it’s bound only to localhost, remote clients cannot connect.

How to do it

  1. From “ports listening”, you already have the list. In ss -tlnp or netstat -tlnp, look at the “Local Address” column: 0.0.0.0:8080 means all interfaces; 127.0.0.1:8080 means localhost only.
  2. Find the server’s config (e.g. bind_address, listen, host). Set it to 0.0.0.0 or the correct interface IP if you need remote access. Restart the service after changing.
