How to check config syntax & paths

Instruction: validate config files and paths referenced in them

What this is

Invalid syntax (e.g. wrong YAML/JSON, unclosed quotes) or wrong paths (missing dirs/files) in config cause the service to fail on startup. You need to validate syntax and ensure every path exists.

How to do it

  1. Locate the config file(s) from the service unit (EnvironmentFile, ExecStart) or the app docs.
  2. If YAML: use a validator or a quick python -c "import yaml; yaml.safe_load(open('config.yml'))". If JSON: python -c "import json; json.load(open('config.json'))" or an online validator.
  3. Search the config for paths (log dir, data dir, certs). Check each with ls -la <path> or Test-Path (PowerShell). Create dirs or fix paths as needed.
