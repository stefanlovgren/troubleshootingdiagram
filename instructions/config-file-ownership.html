How to check config file ownership

Instruction: ensure the service user can read config files

What this is

Config files (e.g. under /etc or the app directory) must be readable by the user the service runs as. “Permission denied” when reading config is often ownership or mode.

How to do it

  1. Find config files from the server docs or under /etc/<app>, <install-dir>/conf. Run ls -la <config-file>.
  2. If owner is root and the service runs as appuser, either make the file readable by that user (e.g. chmod 644 and ensure group or others can read) or change owner: chown appuser:appgroup /etc/app/config.yml. Avoid making config world-writable.
