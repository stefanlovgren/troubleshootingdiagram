How to check run-as user / permissions

Instruction: verify the user running the installer has access

What this is

The installer runs as a specific user. If that user cannot write to the install directory or run certain commands, the install fails with permission errors.

How to do it

  1. Confirm which user you are: whoami (Linux) or echo %USERNAME% (Windows cmd).
  2. Check write access to the target directory: e.g. ls -la /opt and ensure your user (or group) can create and write there. Use sudo only if the docs say to run the installer as root.
  3. If the installer must run as root, run it with sudo or from a root session. If it must run as a dedicated user, create that user first and run the installer logged in as that user (or via su).
