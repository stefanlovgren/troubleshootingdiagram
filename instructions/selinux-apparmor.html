How to check SELinux / AppArmor context

Instruction: verify and fix mandatory access control

What this is

On RHEL/CentOS/Fedora (SELinux) or some Debian/Ubuntu (AppArmor), policy can block the server from reading files, binding ports, or executing binaries even when ownership looks correct.

How to do it

  1. SELinux: Check status with getenforce. If Enforcing, check logs: ausearch -m avc -ts recent or /var/log/audit/audit.log for “denied”. Restore context on app dirs: restorecon -Rv /path/to/app. If the app needs a custom policy, use setsebool or create a module; temporarily test with setenforce 0 (not for production).
  2. AppArmor: aa-status to see loaded profiles. Check /var/log/syslog for AppArmor denials. Put the server in complain mode for its profile to log without blocking: aa-complain /etc/apparmor.d/<profile>, or fix the profile and reload.
