How to check ping / DNS resolution

Instruction: verify basic connectivity and name resolution

What this is

Before digging into firewall or ports, confirm that the server can reach other hosts (or be reached) at the network layer and that hostnames resolve correctly.

How to do it

  1. Ping: From the server, run ping -c 3 <other-host-ip-or-name>. From another machine, ping the server’s IP. If ping fails, the problem is connectivity or firewall (ICMP may be blocked), not the app.
  2. DNS: Run nslookup <hostname> or dig <hostname> to see if the hostname resolves. Check /etc/resolv.conf (Linux) for DNS servers. Fix DNS or use IP if resolution fails.
