How to check directory ownership & mode

Instruction: fix ownership and permissions on data/log dirs

What this is

The server needs to read and often write to its install, data, and log directories. If those are owned by root or have restrictive modes, the run-as user cannot access them.

How to do it

  1. Identify the run-as user (see “Service / run-as user”). List the dirs the app uses: ls -la /path/to/app, ls -la /var/log/<app>, etc.
  2. Owner and group should allow the service user to read (and write where needed). Typical: chown -R appuser:appgroup /path/to/data /path/to/logs.
  3. Set mode so the owner has rwx and group/others as needed, e.g. chmod -R 755 /path/to/data or 750 for more restriction. Log dirs usually need 775 or 755 so the app can write.
