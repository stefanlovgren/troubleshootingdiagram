How to check env vars / defaults

Instruction: verify environment variables the service uses

What this is

Many servers take settings from environment variables (e.g. JAVA_HOME, CONFIG_PATH). If those are wrong or unset when the service starts, the app can fail or use wrong config.

How to do it

  1. systemd: In the unit file, check Environment= and EnvironmentFile=. Use systemctl show <unit> --property=Environment to see resolved values. Ensure any file in EnvironmentFile= exists and contains the expected vars.
  2. Compare with the app’s documentation: required vars, correct format (paths, URLs). Set missing or wrong vars in the unit or the env file, then systemctl daemon-reload and restart the service.
