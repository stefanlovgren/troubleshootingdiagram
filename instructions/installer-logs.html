How to check installer logs

Instruction: find and read the installer log to get the exact error

What this is

The installer usually writes a log file. That log contains the exact error (e.g. permission denied, missing file, failed step) and is the fastest way to see why the install failed.

How to do it

  1. Check the installation guide for “log file location” or “troubleshooting”. Common places: /var/log/<product>, ~/.installer/logs, or a path shown at the end of the installer run.
  2. Open the most recent log file (by date/time). Search for ERROR, FATAL, Exception, or failed.
  3. Read the lines around the error to see which step failed and what resource (file, port, user) was involved. Use that to fix the cause (permissions, disk, dependency) and retry.
