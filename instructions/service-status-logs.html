How to check service status / unit logs

Instruction: see if the service is running and read its logs

What this is

When a server component won’t start or crashes, the first step is to see its state (active, failed, etc.) and read the logs (e.g. systemd journal) for the exact error.

How to do it

  1. Linux (systemd): systemctl status <unit> to see state and last lines of log. For full logs: journalctl -u <unit> -n 100 --no-pager, or journalctl -u <unit> -f to follow. Use journalctl -u <unit> -p err -n 50 to filter by priority.
  2. Windows: Open Services (services.msc) and check status of the service, or Get-Service -Name <Name>. View Event Viewer → Windows Logs → Application for application errors.
