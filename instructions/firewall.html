How to check firewall / security groups

Instruction: verify firewall and cloud security group rules

What this is

Traffic can be blocked by the host firewall (iptables, firewalld, Windows Firewall) or by cloud security groups. The server may need inbound/outbound rules for specific ports or IPs.

How to do it

  1. Linux (firewalld): firewall-cmd --list-all to see zones and rules. Open the required port, e.g. firewall-cmd --permanent --add-port=8080/tcp, then firewall-cmd --reload.
  2. Linux (iptables): iptables -L -n to list rules. Add an ACCEPT rule for the app’s port if it’s currently DROP/REJECT.
  3. Cloud: In the cloud console, open the security group attached to the server and ensure inbound (and outbound if needed) allow the correct port and source (e.g. 0.0.0.0/0 or your VPN).
