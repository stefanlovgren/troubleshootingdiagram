Services & config troubleshooting

When a server component won’t start or misbehaves: check service status and unit logs (e.g. journalctl -u <unit>), then config syntax and paths, then environment variables and defaults, and that any dependent services are up and started in the right order. Restart only after fixing config or dependencies.

This sub-chart covers the “Services & config” node from the main troubleshooting chart. For install or network issues, use Installation failures or Network issues.
