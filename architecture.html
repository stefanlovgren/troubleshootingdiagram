Installation failure troubleshooting

When the server installation fails, follow the flow above: verify disk space, then dependencies (missing packages, wrong versions), then installer logs for the exact error, then permissions of the user running the installer. Retry or perform a clean install only after checking logs.

This sub-chart covers the “Installation failures” node from the main troubleshooting chart. Next check Network issues, Permissions, or Services & config if the install succeeds but the server still misbehaves.
